## How to read these results

Each dataset runs every task twice: **control** (Claude Code alone) vs **treatment** (Claude Code + codemap). Both sessions explore the same repo copy in read-only mode (`--permission-mode plan`) and report which files are relevant to the task.

| Metric | Meaning |
|--------|---------|
| Recall | Fraction of expected files that Claude found |
| Precision | Fraction of files Claude reported that were actually relevant |
| Tool calls | Average number of tool invocations (Read, Grep, Glob, Bash, etc.) |
| Tokens | Total input + output tokens consumed |
| Wall time | Clock time per session |

Negative deltas on tool calls, tokens, and wall time mean the treatment was more efficient. Positive deltas on recall and precision mean it found better results.

**Note:** Treatment sessions use `codemap setup` with heuristic-only metadata (no LLM enrichment). Results may improve further with `codemap enrich --api`.
